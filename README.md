# 🦷 Saudiclinic — نظام إدارة عيادة الأسنان

نظام متكامل لإدارة عيادة أسنان (صفحة واحدة) بواجهة عربية، مبني على HTML/JS خالص
بدون أي Framework، ومتصل بقاعدة بيانات **Supabase** للحفظ والمزامنة عبر الأجهزة،
ومستضاف على **Vercel**.

---

## ✨ المميزات

- **المرضى:** ملف مريض شامل 360° (بيانات + مواعيد + حالات + فواتير + مديونية) مع أزرار سريعة.
- **المواعيد:** حجز وإدارة، وتحويل الموعد إلى حالة بضغطة واحدة.
- **الحالات والخدمات:** تسجيل العلاج، حساب نصيب الطبيب والصافي، كتالوج خدمات وباقات.
- **المالية:**
  - خزينة ووسائل دفع متعددة مع عمولات.
  - فواتير مع دفع جزئي ومتابعة المتبقي.
  - **دفعات بعملات متعددة** (جنيه / ريال / دولار) مع تسجيل **سعر الصرف يوم الدفع**،
    والقيمة تُخزَّن بالجنيه (العملة الأساس) لضمان دقّة المحاسبة.
  - **عرض تقديري** لقيمة الفاتورة بعملة أجنبية.
  - محاسبة وأرباح، حسابات الأطباء، الشركاء، المصروفات.
- **📑 عروض الأسعار / الفواتير التوضيحية:** اختيار العملة، بنود بأسعار يدوية وكميات،
  خصم لكل بند أو على كامل الفاتورة، **خطة أقساط بتواريخ**، قالب احترافي، طباعة،
  حفظ PDF، وإرسال عبر **واتساب**.
- **التسويق:** واتساب، حملات، رسائل SMS.
- **الإدارة:** تقارير، إشعارات، هوية العيادة (شعار + ألوان)، مستخدمون، إعدادات، نسخ احتياطي.
- **مصادقة آمنة** عبر Supabase Auth، وقاعدة بيانات مقفولة على المستخدمين المسجّلين فقط.

---

## 📁 ملفات المشروع

| الملف | الوصف |
|---|---|
| `index.html` | التطبيق بالكامل (الواجهة + المنطق). |
| `supabase.js` | الاتصال بـ Supabase ودوال الحفظ/التحميل + مزامنة الحالة الكاملة (snapshot). |
| `00_setup_all.sql` | إنشاء كل الجداول والصلاحيات (يُشغَّل مرة واحدة). |
| `01_auth_profiles.sql` | جدول `profiles` والإنشاء التلقائي للملف الشخصي عند التسجيل. |
| `02_lockdown_auth.sql` | القفل الأمني النهائي (يمنع الوصول المجهول) — يُشغَّل أخيراً. |

---

## 🚀 خطوات التشغيل (لأول مرة)

### 1) قاعدة البيانات (Supabase ▸ SQL Editor)
شغّل الملفات **بالترتيب**:
1. `00_setup_all.sql`
2. `01_auth_profiles.sql`
3. أنشئ مستخدم Admin من **Authentication ▸ Users ▸ Add user** (مع Auto Confirm).
4. تأكد من تسجيل الدخول وأن البيانات تُحمَّل وتُحفظ بنجاح.
5. أخيراً: `02_lockdown_auth.sql` (القفل الأمني).

> للتراجع وقت الطوارئ: أعد تشغيل `00_setup_all.sql` لإرجاع الوصول مؤقتاً.

### 2) الاستضافة (Vercel)
- الريبو موصول بـ Vercel، وأي رفع جديد لـ `index.html` يعمل Deploy تلقائياً.
- بعد أي تحديث: افتح الموقع واعمل **Hard Refresh** (`Ctrl/Cmd + Shift + R`).

## 🔧 ملاحظات تقنية

- **الحفظ والمزامنة:** كل بيانات التطبيق تُحفظ كنسخة كاملة (snapshot) في جدول `app_state`
  عبر `cloudSaveState()`. لذلك إضافة أي بيانات جديدة في التطبيق **لا تتطلب أي تعديل في قاعدة البيانات**.
- **العملة الأساس:** الجنيه المصري (EGP). الدفعات بعملات أخرى تُحوَّل بسعر يوم الدفع وتُخزَّن بالجنيه.
- لا يحتاج المشروع أي خطوة build — ملفات ثابتة تُخدَّم مباشرة.

---

## 🔐 الأمان

- لا توجد كلمات مرور مخزّنة داخل الكود؛ المصادقة عبر Supabase Auth.
- بعد تشغيل `02_lockdown_auth.sql` لا يمكن قراءة/كتابة البيانات إلا بحساب مسجَّل.
